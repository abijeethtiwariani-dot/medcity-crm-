## How the build works (important for edits)
- New features are appended as `<style id="vNNStyle">` + `<script id="vNNScript">` blocks **before `</body>`** (classic scripts share one global scope — read earlier top-level consts like `titles`, `window.__DB_MASTER`, `window.DB1000`, `window.COUNS`, `window.setVertical`, `window.go`; never redeclare with `const`/`let` at top level if it collides; guard reuse with `typeof X!=='undefined'`). Wrapping `window.go` / `window.setVertical` is the standard pattern for "do X when the user navigates/switches vertical" — always call the previous (`_go`/`_sv`) through, never replace it outright, so the whole chain composes.
- Pages are `.page` divs toggled by `go(p,btn)` via the `titles` map; nav buttons are `.nav-btn[data-page]` inserted into `.side`.
- Data model: `window.__DB_MASTER` = the full immutable lead array (Academy + Study Abroad, id format `L-####`); `window.DB1000` = live vertical-filtered view spliced in place by `setVertical`; each lead tagged `vertical: 'Academy' | 'Study Abroad'`. **There is a second, older, vertical-blind dataset** — the plain `leads` array (~500 records, id format also `L-####` but a *different* id space) that still powers a few legacy pages (dash journeys widget, `leads` kanban, `apps`, `visa`, `student` 360, `reports`, `flow`, `followups`). Don't assume the two datasets share ids — check which one a given page actually reads before wiring anything new to "leads."
- Branch names: DB1000/`__DB_MASTER` use full names like `"Medcity Kochi"`, `"Medcity Thiruvalla KSRTC"` (some records have a trailing `" Branch"` — data hygiene quirk, not fixed everywhere); some legacy code (`COUNS`, the old `leads` array) uses short names (`"Kochi"`, `"Calicut"`) where Calicut ≠ Kochi's official name mapping (Calicut = Kozhikode). Use the `brMatch`/`stripBr` pattern from V11/V28 in this file if you need to cross-reference the two.
- Validate JS with the node syntax-check pattern used throughout this session (regex over `<script>` blocks, skip `type="text/html"`, run `new Function(code)`) before inserting; always keep a `.BEFORE-*.html` backup.
- Preview via a Bash-launched `python3 -m http.server` (not the sandboxed preview_start managed process — that one hit a permissions issue serving `~/Desktop` in this session) + `preview_eval`'s `window.location.href=` to navigate an existing browser tab to it.
